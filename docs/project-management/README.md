# Project Management

Use this folder for planning, scope control, delivery tracking, and project decisions that are not part of the app's developer documentation.

## Planning Documents

- [Project charter](project-charter.md)
- [Requirements](requirements.md)
- [Roadmap](roadmap.md)
- [Milestones](milestones.md)
- [Backlog](backlog.md)
- [Risks and assumptions](risks-and-assumptions.md)
- [Meeting notes](meeting-notes/)

## Suggested Workflow

1. Define the project charter.
2. Write the first version of requirements.
3. Break the release into milestones.
4. Track features and tasks in the backlog.
5. Update risks and assumptions as scope changes.
